Projet BDD

Le projet consiste à partir d’un jeu de données, à réaliser une application web permettant de passer des requêtes et de visualiser les réponses.

Timeline

  • dès maintenant :

    • remplir l’index des dépots git

    • récupérer les données

  • pour le 19/3 : concevoir le modèle de données et le déposer dans le dépot git

Sujet

Il s’agit de concevoir un modèle de données, de l’implémenter pour le SGBD postgres, de peupler la base et de réaliser une application web permettant à un utilisateur non informaticien de l’interroger.

Les données sont un ensemble de boîtes mails (un répertoire par boîte) et un fichier XML.

L’application doit permettre de visualiser simplement les résultats obtenus par les interrogations ci-dessous.

Employés

Sortir l’identité et la catégorie des employés :

  • communiquant le plus en interne

  • les plus prompts à répondre

  • les plus grands diffuseurs d’information (envoi de mail sans réponse)

Ces requêtes seront paramétrables dans l’interface web au niveau :

  • des dates de début et de fin (de la période étudiée)

  • du nombre de valeurs souhaité dans le résultat

  • du seuil (tous les résultats < ou > au seuil)

Couples d’employés

Sortir les paires d’employés (identité et catégorie) communiquant le plus ensemble (avec les mêmes possibilité de paramétrage que ci-dessus).

Jours

Sortir les jours ayant connus le plus grand nombre d’échanges de mails.

Paramètres :

  • début et fin de période

  • nombre de jours souhaité dans le résultat

  • seuil (nombre minimum de mails)

Profil d’employés

Sortir les caractéristiques d’un employé identifié par son nom :

  • son identité et sa catégorie

  • le nombre moyen de mail envoyé (respectivement reçu) par jour

  • le temps de réponse moyen aux mails

  • le rapport nombre de mails internes/mails externes

  • ses contacts internes (les destinataires de ses mails)

Pas de paramétrage
