Organisation du code source

Bien qu’il soit possible de tout concentrer dans un fichier source unique, il est indispensable, dans la pratique, d’écrire un code modulaire et bien organisé, ne serait-ce pour s’y retrouver, soi-même.

Pour cela, on isole le code d’une classe (ou d’un ensemble de classe liées entre elles par une logique de dépendance) dans un fichier .py, que l’on importe pour l’utiliser, à l’identique d’un module :

fichier mesclasses.py
class A:  (1)
  def __init__(self):
    ...
1 définition de la classe

Comme pour les modules, on peut placer, dans ce fichier, du code pour tester la classe, montrer comment l’utiliser, ou faire une démo.

Ce code ne doit évidemment pas être exécuté lorsque le fichier est inclus, mais seulement lorsqu’on le lance comme un script. Cela se fait grace à la variable __name__, de la façon suivante :

fichier mesclasses.py
class A:  (1)
  def __init__(self):
    ...

if __name__ == '__main__':
  ...  (1)
1 code de test, demo, exemple…​

Une fois finalisée, on peut utiliser la classe depuis son projet de la façon suivante :

fichier monprojet.py
#!/bin/env python3
import mesclasses

a = mesclasses.A()

ou bien :

fichier monprojet.py
#!/bin/env python3
from mesclasses import A

a = A()
