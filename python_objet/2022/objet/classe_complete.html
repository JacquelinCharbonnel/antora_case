Classes & instances

Une classe définit un type d’objets. C’est un espace de noms qui regroupe des attributs : des variables et des fonctions (appelées méthodes). Chaque objet de ce type, appelé instance possède son propre jeu d’attributs.

Définition d’une classe

Une bonne pratique est de définir une classe dans un fichier source portant son nom. Définir une classe, c’est spécifier ses attributs et ses méthodes :

définition d’une classe
#!/bin/env python3
# fichier Complex.py
import math

class Complex:
  def module(self):
    return math.sqrt(self.x**2 + self.y**2)

  def conjugue(self):
    c = Complex()
    c.x = self.x
    c.y = -self.y
    return c

Ici, la classe Complex se compose de 2 méthodes conjugue et module.

Chaque méthode possède au moins un paramètre destiné à recevoir l’objet pour lequel elle est appelée. Ce paramètres est traditionnellement nommé self. Il donne accès aux attributs et aux autres méthodes de l’objet.

Instanciation d’une classe

C’est l’opération consistant à créer un objet à partir de la classe.

instanciation d’une classe
#!/bin/env python3
# fichier testComplex.py
from Complex import Complex

z = Complex()
z.x = 1
z.y = 2
z2 = z.conjugue()
print(z2.module())

Méthodes __init__

S’il existe une méthode __init__, alors elle est automatiquement appelée à la création de chaque instance. Son but est d’initialiser l’objet nouvellement créé.

#!/bin/env python3
# fichier Complex.py
import math

class Complex:
    def __init__(self,x,y):
      self.x = x
      self.y = y

    def module(self):
      return math.sqrt(self.x**2 + self.y**2)

    def conjugue(self):
      c = Complex()
      c.x = self.x
      c.y = -self.y
      return c
#!/bin/env python3
# fichier testComplex.py
from Complex import Complex

z = Complex()     # TypeError: __init__() takes exactly 3 arguments (1 given)  (1)

z = Complex(1,2)
print(z.module())
1 Il faut maintenant fournir 2 arguments

Symétriquement, s’il existe une méthode __del__, alors elle est automatiquement appelée à la destruction de toute instance. Son but est d’offrir une dernière occasion de nettoyer ce qui peut l’être avant la disparition de l’objet.
Les attributs et méthodes sont publics, c’est-à-dire qu’on peut y accéder depuis l’extérieur de l’objet. Mais préfixer un attribut ou une méthode avec _ signifie par convention qu’on ne doit pas le faire.

Donner un sens aux opérateurs

Pouvoir appliquer les opérateurs standard de python à de nouveaux types d’objet passe par la définition de méthodes particulières.

Une opération binaire   x α y   est interprétée comme   x.__mα__(y) , où __mα__ est la méthode associée à l’opérateur α. Par exemple, z1==z2 est interprété comme z1.__eq__(z2).

Une opération unaire   αx   est interprétée comme   x.__mα__() , où __mα__ est la méthode associée à l’opérateur α. Par exemple, -z est interprété comme z.__neg__().

Opérateurs relationnels

Les méthodes à définir sont : __lt__, __le__, __eq__, __ne__, __gt__ et __ge__.

Par défaut, __ne__ renvoie la négation de __eq__, et n’a donc pas a être personnalisé dans la plupart des cas.
Opérateurs arithmétiques

Les méthodes à définir sont :

__add__, __sub__, __mul__, __truediv__, __floordiv__, __mod__, __pow__, __neg__, etc.

Exemple :

class Complex:
  ...
  def __eq__(self,z):
      return self.x==z.x and self.y==z.y

  def __add__(self,z):
      return Complex(self.x+z.x, self.y+z.y)

  def __mul__(self,z):
      return Complex(self.x*z.x-self.y*z.y, self.x*z.y+self.y*z.x)

z1= Complex(3,4)
z2= Complex(3,5)
z3= Complex(3,4)

print(z1==z2)          # False
print(z1!=z2)          # True
print(z1<z2)           # TypeError: '<' not supported between instances of 'Complex' and 'Complex'
print(z1==z3,z1 is z3) # True False

print(z1+z2)
print(z1*z2)

Opérateurs de modification in place

Ce sont les opérateurs +=, *=, etc. Sur les types prédéfinis,   x α= y   signifie   x = xαy . Cette définition par défaut n’existe pas pour les objets créés par le programmeur, il faut donc les définir explicitement si l’on veut en disposer.

L’opération   x α= y   est interprétée comme   x.__imα__(y) , où __mα__ est la méthode associée à l’opérateur α. Par exemple, x+=y est interprété comme x.__iadd__(y).

Opérateurs entre opérandes de types différents

Il s’agit par exemple de définir des opérations entre complexes et réels (dans cet ordre). Pour cela, il faut tester le type du paramètre (par exemple, avec isinstance), et faire le calcul en conséquence. Exemple :

class Complex:
  # ...
  def __add__(self,z):
    if isinstance(z,Complex):
      return Complex(self.x+z.x, self.y+z.y)
    elif isinstance(z,int) or isinstance(z,float):
      return Complex(self.x+z, self.y)
    else:
      raise RuntimeError("opération non prévue")

z= Complex(1,1)

print(z+z)      # 2+2i
print(z+1)      # 2+i (1)
print(1+z)      # TypeError: unsupported operand type(s) for +: 'int' and 'Complex' (2)
1 pas de surprise, z+1 est interprêté comme z.__add__(1).
2 par contre, erreur pour 1+z, qui est interprêté comme 1.__add__(z), ce qui n’a pas de sens.

Opérateurs avec premier opérande de type prédéfini

On ne peut pas ajouter une méthode à un type prédéfini, donc la technique précédente n’est pas applicable. Pour cela, il existe d’autres méthodes.

Si x est d’un type prédéfini, alors l’opération   x α y   est interprétée comme   y.__rmα__(x) , où __mα__ est la méthode associée à l’opérateur α. Par exemple, 1+z est interprété comme z.__radd__(1).

Question 3

Enrichir la classe Fraction avec les opérateurs arithmétiques classiques.

class Fraction:
  ...
  def __add__(self,r):
    """addition de quelque cose à un fraction"""
    if isinstance(r,Fraction):
        return Fraction(self.n*r.d+self.d*r.n,self.d*r.d)
    if isinstance(r,int) or isinstance(r,float):
        return self+Fraction(r)
    raise RuntimeError("opération non implémentée pour ce type")

  def __neg__(self):
    """opposé d'un fraction"""
    return Fraction(-self.n,self.d)

  def __eq__(self,r):
    """égalité de 2 fractions"""
    return self.n==r.n and self.d==r.d

  def __iadd(self,r):
    """incrémentation inplace d'un fraction"""
    self = self+r

  def __radd__(self,i):
    """addition d'un fraction à quelque chose"""
    return self+i

ce qui permet d’écrire :

p = Fraction(9,15)
q = Fraction(2,3)

print(p+q)      # p.__add__(q)
print(p+1)      # p.__add__(1)
print(-p)       # p.__neg__()
print(p==q)     # p.__eq__(q)
p += q          # p.__iadd__(q)
print(1+p)      # p.__radd__(1)
print(p+"abcd") # p.__add__("abcd")

Attribut et méthode de classe

Attribut de classe

C’est un attribut partagé par toutes les instances de la classe.

Exemple d’attribut de classe
class Complex:
    nb_instances = 0

    def __init__(self,x,y):
      self.x = x
      self.y = y
      Complex.nb_instances += 1

    def __del__(self):
      Complex.nb_instances -= 1
Méthode de classe

C’est une méthode définie au niveau de la classe, donc commune à toutes les instances. Elle n’a donc pas accès aux attributs des instances : son périmètre d’action est restreint aux membres de classe.

Elle ne reçoit pas un objet en premier paramètre, mais la classe de l’instance pour laquelle elle est appelée. Ce paramètre est nommé par convention cls.

Une méthode de classe peut être appelée sans qu’aucune instance existe, en l’appliquant directement à la classe.

Exemple de méthode de classe
class Complex:
    nb_instances = 0

    def __init__(self,x,y):
      self.x = x
      self.y = y
      Complex.nb_instances += 1
      print(f"Complex({self.x},{self.y}) ({id(self)}) créé")
      print(Complex.how_many_instances())

    @classmethod
    def how_many_instances(cls):
      return f"instances vivantes: {cls.nb_instances}"

    def __del__(self):
      Complex.nb_instances -= 1
      print(f"Complex({self.x},{self.y}) ({id(self)}) détruit")
      print(Complex.how_many_instances())

Organisation des sources

Bien qu’il soit possible de tout concentrer dans un fichier source unique, dans la pratique, il est recommandé d’écrire un code modulaire. Pour cela, on isole le code d’une classe dans un fichier .py dédié (à l’identique d’un module) :

fichier mesclasses.py
class A:  (1)
  def __init__(self):
    ...
1 définition de la classe

A la fin de ce fichier, on peut insérer du code pour tester la classe, montrer comment l’utiliser, ou faire une démo (cf les modules). Ce code ne doit évidemment pas être exécuté lorsque le fichier est inclu, mais seulement lorsqu’on lance l’exécution du fichier comme un script. Cela se fait de la façon suivante :

fichier mesclasses.py
class A:  (1)
  def __init__(self):
    ...

if __name__ == '__main__':
  ...  (1)
1 code de test, demo, exemple…​

Une fois finalisé, on peut utiliser la classe depuis son projet de la façon suivante :

fichier monprojet.py
#!/bin/env python3
import mesclasses

a = mesclasses.A()

ou bien :

fichier monprojet.py
#!/bin/env python3
from mesclasses import A

a = A()
