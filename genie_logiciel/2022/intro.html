Pour une application compilée, le code subit, avant toute exécution et une fois pour toute, une traduction en langage machine compréhensible par le processeur de l’ordinateur. Cette traduction, appelée compilation, produit un fichier binaire (contenant donc des instructions pour le processeur). Celui-ci les exécute donc nativement : l’exécution est donc rapide. Le code source n’est pas nécessaire à l’exécution du binaire. Le binaire est spécifique à une architecture matérielle donnée : par exemple, un binaire compilé pour Linux ne fonctionne pas sous Windows, et inversement.
