Gestion des processus

Exemple
int main()
{
  pid_t pid ;
  int ret ;
  pid = fork() ; /* dédoublement : pid reçoit le PID du fils dans le père
                                   pid reçoit 0 dans le fils */
  switch(pid)
  {
    case -1 :
      perror("erreur\n") ;
      return 1 ;

   case 0 :
      puts("[fils] coucou c'est moi!") ;
      return 3  ;

   default :
      printf("[père] le PID de fiston est %d\n",pid) ;
      puts("[père] j'attends pour récupérer le code de retour de fiston") ;
      wait(&ret) ;
      printf("[père] fiston a renvoyé %d\n",WEXITSTATUS(ret)) ;
      return 0 ;
  }
}
Exercice 1

Pour comprendre ce qui se passe dans le programme ci-dessus, on va allonger le temps d’exécution du fils, pour permettre à son père de consulter la liste des process, de façon à voir le fils :

  • en cours d’exécution (p1),

  • après son exécution, mais avant la récupération de son code de retour (p2),

  • après récupération de son code de retour (p3).

Pour cela :

  • ajouter une temporisation de 1s entre les lignes #16 et #17,

  • ajouter entre #20 et #21 l’exécution de la commande :

ps -o pid,ppid,stat,cmd # (p1)

suivie d’une pause de 2s, suivie de la même commande ps (p2),

  • ajouter à nouveau cette même commande ps entre #22 et #23 (p3).

Regarder et interpréter le résultat.

Indications : system, sleep
Exercice 2

Ecrire un programme qui affiche son propre pid, initialise une variable globale à 5, puis se dédouble en 2 processus. Faire en sorte que les processus obtenus :

  1. respectivement incrémentent et doublent la valeur de la variable,

  2. affichent leur pid, et la valeur de la variable.

Conclusion par rapport aux valeurs affichées ?

Indications : getpid
Exercice 3

Normalement, le processus père doit récupérer (fonction wait) le code de retour de chacun de ses fils, sous peine de laisser des zombis.

Ecrire un programme admettant 2 entiers t1 et t2 en argument de la ligne de commande, et qui se dédouble en 2 processus. Après dédoublement, les processus :

  1. écrivent respectivement "[père] mon PID est pid" et "[fils] mon PID est pid",

  2. exécutent la commande ps -o pid,ppid,stat,cmd (fonction system),

  3. écrivent respectivement "[père] je m’endors pour t1 secondes" et "[fils] je m’endors pour t2 secondes",

  4. puis attendent respectivement t1 secondes et t2 secondes (fonction sleep),

  5. exécutent la commande ps -o pid,ppid,stat,cmd, préfixé respectivement de [père] et [fils],

  6. et enfin écrivent respectivement "[père] je termine" (après avoir attendu son fils) et "[fils] je termine"

Tester le cas t1>t2 et t1<t2.

Exercice 4

Ecrire un programme admettant en argument sur la ligne de commande un entier n, qui crée n processus, puis attend leur terminaison en affichant successivement leur identité et leur code de retour au fur et à mesure des terminaisons. Chaque processus fils i :

  1. affiche son pid et celui de son père dans un message indiquant le début de son exécution,

  2. attend i secondes, puis affiche un message indiquant la fin d’exécution,

  3. affiche son pid et celui de son père dans un message indiquant la fin de son exécution,

  4. se termine en renvoyant la valeur i (fonction exit).

Attention à ne pas faire un programme qui s’emballe.
Indications : getppid
wait renvoie -1 quand il n’y a plus rien à attendre
